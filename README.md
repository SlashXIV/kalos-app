# Kalos — Fitness & Nutrition Tracker

A complete offline-first Android fitness and nutrition tracking app built with Kotlin + Jetpack Compose.

## Features

- **Nutrition journaling** — log meals (Breakfast / Lunch / Dinner / Snacks), track calories and macros (protein, carbs, fat), search 300+ bundled foods, create custom foods
- **Workout tracking** — exercise catalog with 100+ exercises, build workout templates, run active workouts with set/rep/weight logging and rest timer, full history
- **Training programs** — browse and activate multi-week training programs (PPL, Upper/Lower, Full Body)
- **Calendar** — monthly view with nutrition and workout activity dots
- **Dashboard** — daily calorie ring, macro bars, quick-action shortcuts
- **Profile & Goals** — onboarding with TDEE/macro calculation (Mifflin-St Jeor), edit profile and nutritional targets

## Setup

### 1. Install Android Studio

Download and install [Android Studio](https://developer.android.com/studio) (Hedgehog 2023.1.1 or newer).

During installation, accept all SDK licenses. When prompted, install:
- Android SDK Platform 35 (API 35, Android 15)
- Android SDK Build-Tools 35.0.0
- Android Emulator
- Intel HAXM (or equivalent for your CPU)

### 2. Set environment variables (optional, for CLI builds)

```bash
# macOS / Linux — add to ~/.bashrc or ~/.zshrc
export ANDROID_HOME=$HOME/Library/Android/sdk   # macOS
export ANDROID_HOME=$HOME/Android/Sdk           # Linux
export PATH=$PATH:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools

# Windows — set in System Properties > Environment Variables
ANDROID_HOME = C:\Users\<YourUser>\AppData\Local\Android\Sdk
```

### 3. Open the project

```
File > Open > select the kcal_tracker folder
```

Android Studio will sync Gradle automatically. Wait for the sync to complete (first sync downloads dependencies ~500 MB).

### 4. Run the app

- **Emulator:** `Tools > Device Manager > Create Virtual Device` → choose a Pixel 6 with API 34+
- **Physical device:** enable USB debugging in Developer Options, connect via USB
- Click **Run ▶** or press `Shift+F10`

## Build from the command line

### Prerequisites

- JDK 17+ in PATH (`java -version` should show 17+)
- Android SDK installed with `ANDROID_HOME` set (see above)
- Gradle wrapper JAR is generated by Android Studio on first sync.
  If missing, run `gradle wrapper` in the project root (requires Gradle 8.9 installed separately),
  or simply open the project in Android Studio once to let it generate the wrapper.

### Commands

```bash
# Debug APK (output: app/build/outputs/apk/debug/app-debug.apk)
./gradlew assembleDebug

# Unit tests
./gradlew test

# Instrumented tests (requires connected emulator or device)
./gradlew connectedAndroidTest

# Full build check
./gradlew build

# Release APK (requires signing config — see below)
./gradlew assembleRelease
```

On Windows, use `gradlew.bat` instead of `./gradlew`.

### Release signing

Create `app/keystore.jks` and add to `app/build.gradle.kts`:

```kotlin
signingConfigs {
    create("release") {
        storeFile = file("keystore.jks")
        storePassword = System.getenv("KEYSTORE_PASSWORD")
        keyAlias = "kalos"
        keyPassword = System.getenv("KEY_PASSWORD")
    }
}
buildTypes {
    release { signingConfig = signingConfigs.getByName("release") }
}
```

## Architecture

```
MVVM + Repository pattern (pragmatic Clean Architecture)

ViewModel  ←→  Repository interface  ←→  Room DAO / Assets
    ↕                                          ↕
StateFlow<UiState>                    Entity ↔ Domain mapper
```

### Layers

| Package | Responsibility |
|---|---|
| `core/database/` | Room entities, DAOs, TypeConverters |
| `core/domain/model/` | Pure Kotlin domain models (no Android deps) |
| `core/domain/repository/` | Repository interfaces |
| `core/domain/usecase/` | Business logic (BMR, TDEE, macro calculation) |
| `core/data/repository/` | Repository implementations, mappers |
| `core/data/seed/` | First-launch JSON seeder |
| `core/ui/` | Design system (theme, colors, typography) + shared composables |
| `feature/` | Screen-level composables + ViewModels |
| `navigation/` | NavHost, Screen routes, bottom nav items |
| `di/` | Hilt modules |

### Key tech

| Concern | Library |
|---|---|
| UI | Jetpack Compose + Material 3 |
| DI | Hilt 2.52 |
| Database | Room 2.6.1 |
| Navigation | Navigation Compose |
| Async | Coroutines + Flow |
| Charts | Vico 1.15.0 |
| Font | Nunito via Google Fonts (downloadable) |
| Serialization | Kotlinx Serialization (seed JSON) |

## Seed data

On first launch, `DatabaseSeeder` reads three JSON files from `assets/` and populates Room:

- `seed_foods.json` — ~300 foods (fruits, vegetables, meats, dairy, grains, legumes, nuts, beverages, supplements…)
- `seed_exercises.json` — 100 exercises (chest, back, shoulders, arms, legs, abs, cardio, mobility…)
- `seed_programs.json` — 3 training programs (PPL 6-day, Upper/Lower 4-day, Full Body 3-day)

Seeding happens only once (when `food` table is empty).

```
├── app/build.gradle.kts          # dependencies, compileSdk=35, minSdk=26
├── gradle/libs.versions.toml     # version catalog

    ├── navigation/               # KalosNavGraph, Screen sealed class

        ├── onboarding/           # 4 screens + ViewModel
        ├── home/                 # HomeScreen + ViewModel

        ├── calendar/             # CalendarScreen + ViewModel
        └── profile/              # ProfileScreen, EditProfile, EditGoals, Settings
```
